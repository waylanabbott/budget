**hearth-budget/src/components/transaction-sheet.tsx:**

'use client' component. Props: `{ open: boolean, onOpenChange: (open: boolean) => void, accounts: AccountRow[], categories: CategoryRow[] }`.

Implementation:
- Uses Sheet component with side="bottom" — this creates a bottom sheet that slides up from the bottom, ideal for mobile thumb interaction
- Sheet content max-height: `max-h-[85vh]` with `overflow-y-auto`
- Title: "New Transaction"
- Form layout (top to bottom, designed for 375px width with no scroll to submit):

  1. **Amount** (FIRST field — per TXNS-02): Large numeric input at the top. Use `<Input type="text" inputMode="decimal" pattern="[0-9]*[.]?[0-9]{0,2}" placeholder="0.00" className="text-3xl font-bold text-center h-16 border-0 border-b-2">`. Parse to number on submit. Auto-focus this field when sheet opens.

  2. **Category** (compact): CategoryPicker component with categories prop. Horizontal scrollable row of recent 6 + "More" expansion.

  3. **Account** (dropdown): HTML `<select>` with non-archived accounts. Default to first account if only one exists.

  4. **Date**: `<Input type="date">` defaulting to today in YYYY-MM-DD format. Use `new Date().toISOString().split('T')[0]` for default — but NOTE: this gives UTC date. For local date use: `const today = new Date(); const yyyy = today.getFullYear(); const mm = String(today.getMonth()+1).padStart(2,'0'); const dd = String(today.getDate()).padStart(2,'0'); return \`${yyyy}-${mm}-${dd}\``

  5. **Merchant**: MerchantAutocomplete component

  6. **Notes**: `<Input placeholder="Notes (optional)">` — single line, not textarea

  7. **Submit button**: Full-width `<Button className="w-full" size="lg">Save Transaction</Button>` — always visible without scrolling on 375px screen

- Use react-hook-form with zodResolver and transactionSchema
- The amount field stores a string in the input but converts to number for validation: use `z.coerce.number()` or manually parseFloat before submission
- On submit: call createTransaction(validatedData), show toast on error, close sheet + reset form on success
- Disable submit button while submitting (useTransition or form.formState.isSubmitting)
