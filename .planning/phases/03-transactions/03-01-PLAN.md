**hearth-budget/src/lib/schemas/accounts.ts:**
```typescript
import * as z from 'zod'

export const ACCOUNT_TYPES = ['checking', 'savings', 'credit_card', 'cash'] as const
export type AccountType = (typeof ACCOUNT_TYPES)[number]

export const accountSchema = z.object({
  name: z.string().min(1, { error: 'Account name is required.' }).max(100, { error: 'Account name must be under 100 characters.' }).trim(),
  type: z.enum(ACCOUNT_TYPES, { error: 'Select a valid account type.' }),
  starting_balance: z.number().min(0, { error: 'Starting balance cannot be negative.' }).multipleOf(0.01, { error: 'Balance must have at most 2 decimal places.' }),
})

export type AccountInput = z.infer<typeof accountSchema>
```

**hearth-budget/src/lib/schemas/categories.ts:**
```typescript
import * as z from 'zod'

export const categorySchema = z.object({
  name: z.string().min(1, { error: 'Category name is required.' }).max(50, { error: 'Category name must be under 50 characters.' }).trim(),
  parent_id: z.string().uuid({ error: 'Invalid parent category.' }).nullable().optional(),
  icon: z.string().max(50, { error: 'Icon name too long.' }).nullable().optional(),
  color: z.string().regex(/^#[0-9a-fA-F]{6}$/, { error: 'Color must be a hex value like #FF5733.' }).nullable().optional(),
  is_income: z.boolean().optional(),
})

export type CategoryInput = z.infer<typeof categorySchema>
```

**hearth-budget/src/lib/schemas/transactions.ts:**
```typescript
import * as z from 'zod'

// Date string in YYYY-MM-DD format (occurred_on is DATE type, never toISOString())
const dateStringSchema = z.string().regex(/^\d{4}-\d{2}-\d{2}$/, { error: 'Date must be YYYY-MM-DD format.' })

export const transactionSchema = z.object({
  amount: z.number()
    .positive({ error: 'Amount must be greater than zero.' })
    .multipleOf(0.01, { error: 'Amount must have at most 2 decimal places.' })
    .max(999999999.99, { error: 'Amount is too large.' }),
  account_id: z.string().uuid({ error: 'Please select an account.' }),
  category_id: z.string().uuid({ error: 'Please select a category.' }).nullable().optional(),
  occurred_on: dateStringSchema,
  merchant: z.string().max(200, { error: 'Merchant name too long.' }).trim().nullable().optional(),
  notes: z.string().max(500, { error: 'Notes must be under 500 characters.' }).trim().nullable().optional(),
})

export type TransactionInput = z.infer<typeof transactionSchema>
```

CRITICAL: amount is always positive; sign is inferred from category.is_income at display time (per TXNS-08). occurred_on must be YYYY-MM-DD string, never a Date object or ISO string.

**hearth-budget/src/app/actions/accounts.ts:**

Functions to export:
1. `getAccounts()` — fetch all accounts for user's household (non-archived first, then archived), return `{ data: Account[], error?: string }`. Include computed balance using a Supabase RPC or a second query: `starting_balance + SUM(CASE WHEN categories.is_income THEN amount ELSE -amount END)` from transactions joined to categories. NOTE: For the initial implementation, return accounts without computed balance (balance computation will happen client-side or in a follow-up), just query `supabase.from('accounts').select('*').eq('household_id', householdId).order('is_archived').order('name')`.
2. `createAccount(input: AccountInput)` — validate with accountSchema.safeParse(), insert into accounts table with household_id, return `{ error?: string }`. Revalidate path '/app/accounts'.
3. `updateAccount(id: string, input: AccountInput)` — validate, update accounts row where id matches, return `{ error?: string }`. Revalidate path '/app/accounts'.
4. `archiveAccount(id: string)` — set is_archived = true, return `{ error?: string }`. Revalidate path '/app/accounts'.

For household_id lookup, use this pattern in every action:
```typescript
const { data: member } = await supabase
  .from('household_members')
  .select('household_id')
  .eq('user_id', user.id)
  .single()
if (!member) redirect('/onboarding')
const householdId = member.household_id
```

Use `import { revalidatePath } from 'next/cache'` to revalidate after mutations.

**hearth-budget/src/app/actions/categories.ts:**

Functions to export:
1. `getCategories()` — fetch all categories for household, ordered by sort_order. Return `{ data: Category[], error?: string }`. Query: `supabase.from('categories').select('*').eq('household_id', householdId).is('archived_at', null).order('sort_order')`.
2. `getAllCategories()` — same but includes archived (for management page). No `archived_at` filter.
3. `createCategory(input: CategoryInput)` — validate with categorySchema.safeParse(), insert with household_id. Get next sort_order via `select('sort_order').order('sort_order', { ascending: false }).limit(1)`. Revalidate '/app/settings/categories'.
4. `updateCategory(id: string, input: CategoryInput)` — validate, update. Revalidate '/app/settings/categories'.
5. `archiveCategory(id: string)` — set archived_at = new Date().toISOString(). Revalidate '/app/settings/categories'.

1. `createTransaction(input: TransactionInput)` — validate with transactionSchema.safeParse(), insert with household_id, entered_by = user.id, source = 'manual'. Return `{ error?: string }`. Revalidate '/app/transactions'.
2. `updateTransaction(id: string, input: TransactionInput)` — validate, update where id and household_id match. Revalidate '/app/transactions'.

5. `searchMerchants(query: string)` — `SELECT DISTINCT merchant FROM transactions WHERE household_id = ? AND merchant ILIKE ?` limited to 10 results. Return `{ data: string[], error?: string }`.

CRITICAL: occurred_on values must be YYYY-MM-DD strings. Never call toISOString() on dates for this field. The source field defaults to 'manual' for user-entered transactions.
