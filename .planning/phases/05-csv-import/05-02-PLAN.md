    Create hearth-budget/src/lib/csv/transform.ts that bridges raw CSV rows to transaction insert objects.

    **IMPORTANT: `transformCsvRows` must be `async` because it calls `generateTransactionHash` which is async (uses Web Crypto API).** The hash function returns `Promise<string>`, so each row's hash must be awaited.

    Export interface and function:

    ```typescript
    import type { ColumnMapping, AmountMode } from '@/lib/schemas/csv-import'
    import { generateTransactionHash, normalizeAmount, parseDateString } from './dedup'

    export interface TransformConfig {
      accountId: string
      columnMapping: ColumnMapping
      amountMode: AmountMode
      dateFormat: 'MM/DD/YYYY' | 'YYYY-MM-DD' | 'M/D/YYYY' | 'DD/MM/YYYY'
    }

    export interface TransformedRow {
      row_index: number
      occurred_on: string          // YYYY-MM-DD
      amount: number               // signed: negative=expense, positive=income
      merchant: string | null
      category_hint: string | null // raw category string from CSV (for auto-suggestion lookup)
      notes: string | null
      external_hash: string
      error: string | null         // non-null if this row failed to parse
    }

    export interface TransformResult {
      rows: TransformedRow[]
      errors: Array<{ row_index: number; error: string }>
    }

    // ASYNC — calls generateTransactionHash which uses Web Crypto API
    export async function transformCsvRows(
      csvRows: string[][],
      config: TransformConfig
    ): Promise<TransformResult>
    ```

    Implementation of `transformCsvRows`:
    - Iterate over csvRows (each is a string[] of cell values)
    - For each row:
      1. Extract date cell at `config.columnMapping.date` index. Call `parseDateString(cell, config.dateFormat)`. If null, add to errors with "Invalid date" and skip row.
      2. Extract amount: if `config.amountMode === 'debit_credit'`, get cells at `config.columnMapping.debit` and `config.columnMapping.credit` indices, call `normalizeAmount('', config.amountMode, debitCell, creditCell)`. Otherwise get cell at `config.columnMapping.amount` index, call `normalizeAmount(cell, config.amountMode)`. If null, add to errors with "Invalid amount" and skip row.
      3. Extract merchant: cell at `config.columnMapping.merchant` index if defined. Trim. Set to null if empty string.
      4. Extract category_hint: cell at `config.columnMapping.category` index if defined. Trim. Set to null if empty.
      5. Extract notes: cell at `config.columnMapping.notes` index if defined. Trim. Set to null if empty.
      6. Generate external_hash: `await generateTransactionHash(config.accountId, occurred_on, amount, merchant)` -- note the `await` since this is now async.
      7. Push to rows array as TransformedRow (row_index is the 0-based index in csvRows)

    - Return { rows: successfully parsed rows, errors: array of { row_index, error } }

    **Performance note:** For large CSVs (1000+ rows), consider batching hash generation with `Promise.all` on chunks of ~100 rows to avoid blocking the event loop while maintaining reasonable memory usage. A simple approach: process rows sequentially with `for...of` and `await` each hash -- the Web Crypto API is fast enough that sequential awaits are acceptable for typical CSV sizes (under 10k rows).

    Use the same `getHouseholdId()` inline helper pattern from transactions.ts (copy the pattern: createClient, getUser, household_members query, returns {supabase, user, householdId}).

    Export 6 server actions:

    **1. `checkDuplicates(accountId: string, hashes: string[]): Promise<{ duplicateHashes: string[]; error?: string }>`**
    - Call getHouseholdId()
    - Query transactions: `supabase.from('transactions').select('external_hash').eq('account_id', accountId).in('external_hash', hashes)`
    - Return the set of hashes that already exist as `duplicateHashes`
    - This allows the UI to mark rows as duplicates before import

    **2. `suggestCategories(merchants: string[]): Promise<{ suggestions: Record<string, { category_id: string; category_name: string } | null>; error?: string }>`**
    - Call getHouseholdId()
    - For each unique merchant in the input array (deduplicated, case-insensitive):
      - Query: `supabase.from('transactions').select('category_id, categories(name)').eq('household_id', householdId).ilike('merchant', merchant).not('category_id', 'is', null).limit(1).order('created_at', { ascending: false })`
      - This finds the most recent transaction with the same merchant that has a category assigned
    - Build a Record mapping each merchant to its suggested { category_id, category_name } or null
    - To avoid N+1 queries, batch this: query all transactions matching ANY of the merchants:
      ```typescript
      const uniqueMerchants = [...new Set(merchants.map(m => m.toLowerCase().trim()).filter(Boolean))]
      // For each merchant, find most recent categorized transaction
      const suggestions: Record<string, { category_id: string; category_name: string } | null> = {}
      for (const merchant of uniqueMerchants) {
        const { data } = await supabase
          .from('transactions')
          .select('category_id, merchant, categories(name)')
          .eq('household_id', householdId)
          .ilike('merchant', merchant)
          .not('category_id', 'is', null)
          .order('created_at', { ascending: false })
          .limit(1)
        if (data && data[0] && data[0].category_id) {
          const cat = data[0].categories as { name: string } | null
          suggestions[merchant] = { category_id: data[0].category_id, category_name: cat?.name ?? 'Unknown' }
        } else {
          suggestions[merchant] = null
        }
      }
      ```
    - Limit to max 50 unique merchants to prevent abuse
    - Return { suggestions }

    **3. `executeImport(params: { filename: string; rows: Array<{ occurred_on: string; amount: number; merchant: string | null; category_id: string | null; notes: string | null; external_hash: string }>; account_id: string; skip_duplicate_hashes: string[] }): Promise<{ import_id: string; imported_count: number; skipped_count: number; errors: Array<{ row: number; error: string }>; error?: string }>`**
    - Call getHouseholdId()
    - Filter out rows whose external_hash is in skip_duplicate_hashes
    - Create import log entry: insert into imports table with status='pending', filename, row_count=filteredRows.length
    - Get the import_id from the insert result (use .select('id').single())
    - Build transaction insert objects: for each filtered row, create:
      ```typescript
      {
        household_id: householdId,
        account_id: params.account_id,
        category_id: row.category_id,
        entered_by: user.id,
        amount: row.amount,
        occurred_on: row.occurred_on,
        merchant: row.merchant,
        notes: row.notes,
        source: 'csv' as const,
        external_hash: row.external_hash,
        import_id: importId,
      }
      ```
    - Batch insert all transactions: `supabase.from('transactions').insert(transactionInserts)`
    - If insert succeeds: update imports row to status='complete', row_count=actual inserted count
    - If insert fails (e.g., unique constraint on external_hash): update imports to status='error', set errors jsonb field with the error message. Note: some rows may have been inserted if the batch partially succeeded. For safety, if error occurs, try to get count of inserted rows: `supabase.from('transactions').select('id', { count: 'exact', head: true }).eq('import_id', importId)`
    - Call `revalidatePath('/app/transactions')` and `revalidatePath('/app/import')` on success
    - Return { import_id, imported_count, skipped_count, errors }

    **Note:** `executeImport` receives pre-transformed rows (already hashed). It does NOT call `transformCsvRows` itself. The transform happens client-side in the preview/duplicate-review step so users can see the parsed data before importing. The server action just does the database insert.

    **4. `getImports(): Promise<{ data: Array<{ id: string; filename: string; row_count: number; status: string; errors: unknown; created_at: string }>; error?: string }>`**
    - Call getHouseholdId()
    - Query: `supabase.from('imports').select('id, filename, row_count, status, errors, created_at').eq('household_id', householdId).order('created_at', { ascending: false }).limit(50)`
    - Return { data: rows }

    **5. `getImportDetail(importId: string): Promise<{ import_row: { id: string; filename: string; row_count: number; status: string; errors: unknown; created_at: string } | null; transactions: Array<{ id: string; occurred_on: string; amount: number; merchant: string | null; category_id: string | null; categories: { name: string } | null }>; error?: string }>`**
    - Call getHouseholdId()
    - Query the import row: `supabase.from('imports').select('*').eq('id', importId).eq('household_id', householdId).single()`
    - Query transactions for this import: `supabase.from('transactions').select('id, occurred_on, amount, merchant, category_id, categories(name)').eq('import_id', importId).order('occurred_on', { ascending: false })`
    - Return both

    **6. `undoImport(importId: string): Promise<{ error?: string }>`**
    - Call getHouseholdId()
    - Verify the import belongs to this household: `supabase.from('imports').select('id').eq('id', importId).eq('household_id', householdId).single()`
    - If not found, return { error: 'Import not found.' }
    - Delete the import row: `supabase.from('imports').delete().eq('id', importId)`
    - The ON DELETE CASCADE on transactions.import_id will automatically delete all associated transactions
    - Call `revalidatePath('/app/transactions')` and `revalidatePath('/app/import')`
    - Return {} on success
