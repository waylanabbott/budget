    **2. Create hearth-budget/src/lib/csv/parser.ts:**

    A pure function that takes CSV text and returns headers + rows. No dependencies needed -- use simple RFC 4180 parsing (handles quoted fields with commas and newlines).

    ```typescript
    export interface CsvParseResult {
      headers: string[]
      rows: string[][]
      rawRowCount: number
    }

    export function parseCsvText(text: string): CsvParseResult
    ```

    Implementation:
    - Trim the text
    - Split into lines handling both \r\n and \n
    - Parse each line respecting double-quoted fields (a quoted field can contain commas and newlines)
    - First non-empty line is headers (trimmed, lowercased)
    - Return { headers, rows (as string[][] where each inner array is cell values), rawRowCount }
    - Skip completely empty rows

    The parser must handle:
    - Quoted fields: `"some, value"` -> `some, value`
    - Escaped quotes: `"He said ""hi"""` -> `He said "hi"`
    - Empty fields: `,,` -> `['', '', '']`
    - Trailing commas on rows
    - BOM character at start of file (strip ﻿)

    **3. Create hearth-budget/src/lib/csv/bank-formats.ts:**

    Define bank format signatures and default column mappings.

    ```typescript
    export interface BankFormat {
      name: string
      id: string
      headerSignature: string[]  // lowercase headers that uniquely identify this bank
      mapping: ColumnMapping
      amountMode: AmountMode
      dateFormat: 'MM/DD/YYYY' | 'YYYY-MM-DD' | 'M/D/YYYY' | 'DD/MM/YYYY'
    }

    export const BANK_FORMATS: BankFormat[] = [...]

    export function detectBankFormat(headers: string[]): BankFormat | null
    ```

    Bank format definitions (based on real CSV exports):

    a) **Chase** (credit card): Headers include "transaction date", "post date", "description", "category", "type", "amount". Amount is single signed (negative = charge, positive = payment/credit). Date format: MM/DD/YYYY.
       - mapping: { date: indexOf("transaction date"), amount: indexOf("amount"), merchant: indexOf("description"), category: indexOf("category") }
       - amountMode: 'single_signed'

    b) **Chase** (checking): Headers include "details", "posting date", "description", "amount", "type", "balance", "check or slip #". Amount is single signed. Date: MM/DD/YYYY.
       - mapping: { date: indexOf("posting date"), amount: indexOf("amount"), merchant: indexOf("description") }
       - amountMode: 'single_signed'

    c) **Bank of America**: Headers include "date", "description", "amount", "running bal." Amount is single signed (negative = debit). Date: MM/DD/YYYY.
       - mapping: { date: 0, merchant: 1, amount: 2 }
       - amountMode: 'single_signed'

    d) **Wells Fargo**: Headers often are just 5 columns with no header row, OR headers: "date", "amount", "star", "blank", "description". Actually Wells Fargo exports vary. Use signature: first row has 5 columns and col 0 looks like a date. More reliably: headers might be empty. For well-known pattern: `"date","amount","*","","description"`.
       - mapping: { date: 0, amount: 1, merchant: 4 }
       - amountMode: 'single_signed'

    e) **Capital One**: Headers include "transaction date", "posted date", "card no.", "description", "category", "debit", "credit". Uses separate debit/credit columns. Date: YYYY-MM-DD.
       - mapping: { date: indexOf("transaction date"), merchant: indexOf("description"), category: indexOf("category"), debit: indexOf("debit"), credit: indexOf("credit") }
       - amountMode: 'debit_credit'

    f) **Apple Card**: Headers include "transaction date", "clearing date", "description", "merchant", "category", "type", "amount (purchase/adjustment)". Single signed. Date: MM/DD/YYYY.
       - mapping: { date: indexOf("transaction date"), amount: indexOf("amount (purchase/adjustment)") OR the column containing "amount", merchant: indexOf("merchant") OR indexOf("description"), category: indexOf("category") }
       - amountMode: 'single_signed'

    g) **Discover**: Headers include "trans. date", "post date", "description", "amount", "category". Single signed (positive = charge). Date: MM/DD/YYYY.
       - mapping: { date: indexOf("trans. date"), amount: indexOf("amount"), merchant: indexOf("description"), category: indexOf("category") }
       - amountMode: 'single_unsigned_expense'

    h) **AMEX**: Headers include "date", "description", "amount". Also might include "card member", "account #". Single column, positive = charge. Date: MM/DD/YYYY.
       - mapping: { date: indexOf("date"), amount: indexOf("amount"), merchant: indexOf("description") }
       - amountMode: 'single_unsigned_expense'

    i) **Mint**: Headers include "date", "description", "original description", "amount", "transaction type", "category", "account name", "labels", "notes". Amount is always positive. The "transaction type" column has "debit" or "credit". Date: M/D/YYYY.
       - mapping: { date: indexOf("date"), amount: indexOf("amount"), merchant: indexOf("description"), category: indexOf("category"), notes: indexOf("notes") }
       - amountMode: 'single_unsigned_expense' (but check "transaction type" column -- if "credit" then negate)
       - dateFormat: 'M/D/YYYY'

    The `detectBankFormat(headers: string[])` function:
    - Normalize headers to lowercase and trimmed
    - For each BANK_FORMATS entry, check if ALL items in headerSignature are present in the normalized headers
    - Return the FIRST match (order BANK_FORMATS from most-specific to least-specific signatures)
    - Return null if no match

    Also export a helper:
    ```typescript
    export function buildMappingFromHeaders(headers: string[]): ColumnMapping
    ```
    This is a fallback for unrecognized formats: look for common column names like "date", "amount", "description"/"merchant"/"payee", "category", "debit", "credit", "memo"/"notes" and return a best-guess mapping. Return empty mapping fields as undefined (user must manually map).

    **4. Create hearth-budget/src/lib/csv/dedup.ts:**

    **CRITICAL: Use the Web Crypto API, NOT Node.js `crypto` module.** This file is imported by `transform.ts`, which is called from `csv-preview.tsx` (a `'use client'` component). Node's `crypto` module is NOT available in browser runtimes. The Web Crypto API (`crypto.subtle`) works in both browsers and Node.js >= 15.

    ```typescript
    export async function generateTransactionHash(
      accountId: string,
      date: string,        // YYYY-MM-DD
      amount: number,
      merchant: string | null
    ): Promise<string>
    ```

    Implementation:
    - Normalize merchant: trim, lowercase, take first 20 chars. If null, use empty string.
    - Build a string: `${accountId}|${date}|${amount.toFixed(2)}|${normalizedMerchant}`
    - Hash with the **Web Crypto API** (works in both browser and Node.js):
      ```typescript
      const encoder = new TextEncoder()
      const data = encoder.encode(input)
      const hashBuffer = await crypto.subtle.digest('SHA-256', data)
      const hashArray = Array.from(new Uint8Array(hashBuffer))
      const hashHex = hashArray.map(b => b.toString(16).padStart(2, '0')).join('')
      return hashHex
      ```
    - Do NOT use `import { createHash } from 'crypto'` -- this is Node-only and will throw a runtime error in client components.
    - The `crypto` global (with `crypto.subtle`) is available in all modern browsers and in Node.js >= 15 without any import.
    - Return the hex digest string (Promise<string>).

    Also export:
    ```typescript
    export function normalizeAmount(
      value: string,
      mode: AmountMode,
      debitValue?: string,
      creditValue?: string
    ): number | null
    ```

    Implementation for normalizeAmount:
    - Strip dollar signs, commas, whitespace from value/debitValue/creditValue
    - If mode is 'single_signed': parse as float. Negative means expense (store as negative), positive means income (store as positive). Return the parsed number.
    - If mode is 'single_unsigned_expense': parse as float. Always store as negative (it's an expense). Return `-Math.abs(parsed)`.
    - If mode is 'debit_credit': parse debitValue and creditValue. If debit is present and non-zero, return `-Math.abs(debit)`. If credit is present and non-zero, return `Math.abs(credit)`. If both present, use debit. If neither, return null.
    - Return null if parsing fails (NaN).
    - Round to 2 decimal places: `Math.round(result * 100) / 100`

    Also export:
    ```typescript
    export function parseDateString(
      value: string,
      format: 'MM/DD/YYYY' | 'YYYY-MM-DD' | 'M/D/YYYY' | 'DD/MM/YYYY'
    ): string | null  // returns YYYY-MM-DD or null if invalid
    ```

    Implementation:
    - MM/DD/YYYY: split on '/', month=parts[0], day=parts[1], year=parts[2]
    - YYYY-MM-DD: already correct format, validate and return
    - M/D/YYYY: same as MM/DD/YYYY but parts may be 1 digit
    - DD/MM/YYYY: split on '/', day=parts[0], month=parts[1], year=parts[2]
    - Pad month and day to 2 digits
    - Validate: month 1-12, day 1-31, year 4 digits
    - Return `${year}-${month.padStart(2,'0')}-${day.padStart(2,'0')}` or null if invalid
