    ```typescript
      accountId: string
      filename: string
      categories: Array<{ id: string; name: string; parent_id: string | null }>  // pass from parent
      onImportComplete: (result: { import_id: string; imported_count: number; skipped_count: number; errors: Array<{ row: number; error: string }> }) => void
      onBack: () => void
    }
    ```

    State:
    - `duplicateHashes: Set<string>` -- hashes flagged as duplicates
    - `skipHashes: Set<string>` -- hashes user chose to skip (starts as copy of duplicateHashes)
    - `categorySuggestions: Record<string, { category_id: string; category_name: string } | null>`
    - `categoryOverrides: Record<number, string>` -- row_index -> chosen category_id (user can change)
    - `loading: boolean`
    - `importing: boolean`

    On mount (useEffect):
    - Extract unique hashes from rows: `rows.map(r => r.external_hash)`
    - Call `checkDuplicates(accountId, hashes)` server action
    - Set duplicateHashes and skipHashes from result
    - Extract unique non-null merchants: `[...new Set(rows.map(r => r.merchant).filter(Boolean))]`
    - Call `suggestCategories(merchants)` server action
    - Set categorySuggestions from result
    - Set loading=false

    UI:
    - If loading: show Skeleton placeholders
    - Summary bar: "{rows.length} rows ready, {duplicateHashes.size} duplicates found, {errors.length} rows with errors"
    - If duplicates exist: show a section "Duplicate Transactions" with a list of the duplicate rows. Each row shows: occurred_on, amount, merchant, and a checkbox "Skip this duplicate" (checked by default). User can uncheck to import anyway.
    - Rows table: show ALL rows (not just first 10) in a scrollable container with max-height. Columns: Row#, Date, Amount, Merchant, Category, Status. Category column shows: if categorySuggestions has a match for this merchant, show the suggested category name with a small "auto" badge. User can click to override with a native select of all categories. If no suggestion, show "Uncategorized" in muted text. The Status column shows: green "New" for non-duplicate rows, yellow "Duplicate" with skip checkbox for duplicates, red "Error: {message}" for error rows.
    - For practical UI with potentially hundreds of rows: show summary counts at top, only render full table if < 200 rows, otherwise show "Showing first 200 of {N} rows" with a note that all rows will be imported.
    - Action buttons: "Back" (secondary), "Import {count} Transactions" (primary, where count = rows.length - skipHashes.size). Button shows loading spinner when importing.
    - On "Import" click:
      1. Set importing=true
      2. Build rows for executeImport: for each row NOT in skipHashes, create: `{ occurred_on, amount, merchant, category_id: categoryOverrides[row_index] ?? categorySuggestions[merchant]?.category_id ?? null, notes, external_hash }`
      3. Call `executeImport({ filename, rows: importRows, account_id: accountId, skip_duplicate_hashes: [...skipHashes] })`
      4. On success: call onImportComplete with the result
      5. On error: show toast error
      6. Set importing=false
